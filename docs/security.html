Security & Audit Status

How we think about safety, what’s live today, and what will be audited before major upgrades (staking/perks).

Current status

  • Token contract: Live on BNB Chain
  • Planned: Payments integrations + staking-perks contract(s)
  • Audit: Not yet completed (this page tracks progress)

Contract: 0x16C7eE66633586f781fc47960221DC1bF108906F

Threat model (plain English)

  • Fake tokens / phishing: users are tricked into buying an impostor contract address.
  • DEX risks: price impact and slippage, MEV, and low-liquidity manipulation.
  • Admin key risks: if future contracts have admin controls, compromised keys can harm users.
  • Staking contract risks: bugs can lock funds or allow unfair withdrawals if not audited.

Audit plan (what we will do)

  1. Publish all smart contract source code in a public repo.
  2. Tag a release (e.g., v1.0.0) that matches the deployed bytecode.
  3. Run automated analysis (Slither/static checks) and fix findings.
  4. Engage a reputable auditor for any staking / treasury / vesting contracts.
  5. Publish the audit report here and track fixes.

We will not claim “audited” until a third-party report is published.

Security is a process. If you find an issue, contact us via the official X account.
